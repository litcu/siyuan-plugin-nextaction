# 今天干点啥（NextAction）

面向 [思源笔记](https://b3log.org/siyuan/) 的 GTD 任务管理插件。

它的核心是帮你决定下一步该做什么。你可以把任意块转成任务，补上截止日、重要性、工作量、上下文等信息，插件会据此筛出当前能推进的事，并排出一个可执行的顺序。任务仍然留在思源笔记里，适合已经在这里写项目笔记、会议记录和灵感清单的人。

<!-- screenshot -->

## 快速上手

1. 从思源集市安装「今天干点啥」，也可以下载 Release 后放到工作空间的 `data/plugins/` 目录。
2. 在任意文档里输入 `/zrw`，把当前块转为任务。
3. 点击块左侧的状态圆圈，设置状态、优先级，或打开任务详情。
4. 打开右侧面板，看「下一步行动」「我的一天」「收集箱」。

先从这几步用起来就够了。重要性、回顾、提醒、依赖这些细项，可以等任务变多以后再慢慢加。

## 它怎么配合 GTD

NextAction 按 GTD 的几个动作组织：先收集，再明确，之后放进合适的清单，定期回顾，最后只看当前能执行的事。

### 收集：先记下来

- 输入 `/zrw` 把当前块转成任务，默认进入**收集箱**。
- 输入 `/zrxm` 把当前块转成项目。
- 输入 `/zrwz` 批量转换当前列表或文档子块。
- 也可以右键块图标或文档标题图标，选择「转为任务」。

收集箱里的东西不要求马上想清楚。先放进去，别让它在脑子里占位置。

### 明确：决定下一步

打开**收集箱**视图，逐条处理：

- 要做的，设为「待开始」或「进行中」。
- 暂时不做的，放到「将来/也许」。
- 卡在别人或外部条件上的，设为「等待中」。
- 已经没有价值的，直接移除任务属性。

需要更多信息时，可以在任务详情里补截止日、开始日、重要性、工作量、上下文、标签、备注等字段。

### 组织：放到正确的位置

六种状态对应不同清单：

| 状态 | 图标 | 适合放什么 |
|------|------|------------|
| 收集箱 | 浅蓝圆圈 + 下箭头 | 刚捕获，还没处理 |
| 待开始 | 灰色空心圆 | 已经明确，但还没动手 |
| 进行中 | 蓝色半填充圆 | 正在推进 |
| 等待中 | 橙色虚线圆 | 等人、等反馈、等条件 |
| 将来/也许 | 金色圆 + 三点 | 以后可能会做 |
| 已完成 | 绿色实心圆 + 白勾 | 已完成 |

优先级有五档：紧急（红）> 高（橙）> 中（蓝）> 低（灰）> 无。

任务可以嵌套。项目下面可以有任务，任务下面也可以继续拆子任务。只要子任务没做完，父任务就不会被当成「下一步行动」推到前面。

### 回顾：别让清单沉底

- **回顾**视图会列出一份 GTD 风格的检查清单，也会显示到期需要回顾的任务。
- 每个任务都可以设置回顾间隔，到时间后重新出现在待回顾列表。
- 回顾时可以顺手把「等待中」改回「待开始」，或把「将来/也许」重新激活。

### 执行：只看现在能做的

**下一步行动**只保留当前可以推进的任务。已完成、等待中、未到开始日期、还有未完成子任务的任务，都会被过滤掉。

如果要安排一天的节奏，可以用**我的一天**：从下一步行动里挑任务，拖到时间线上排期。侧边栏里也有紧凑版入口，适合边写笔记边看任务。

## 视图

| 视图 | 用途 |
|------|------|
| 下一步行动 | 当前能做的任务，按计算得分排序 |
| 全部任务 | 未完成任务的树形总览，支持搜索、筛选和拖拽排序 |
| 项目视图 | 按项目分组查看子任务和完成进度 |
| 收集箱 | Inbox 状态任务，集中做明确处理 |
| 我的一天 | 今日计划，可用列表或时间线模式 |
| 将来/也许 | 暂时搁置的任务，可重新激活 |
| 等待中 | 依赖别人或外部条件的任务 |
| 回顾 | GTD 检查清单和待回顾任务 |
| 统计 | 完成率、状态分布等概览 |
| 提醒 | 待处理提醒集中查看 |

## 任务属性

| 类别 | 字段 |
|------|------|
| 基础 | 标题、状态、优先级 |
| 日期 | 截止日、开始日（可精确到分钟） |
| 优先级参数 | 重要性（1-7）、工作量（1-7） |
| 组织 | 上下文（@）、标签、父任务、项目 |
| 依赖 | 依赖任务、依赖模式（全部/任一）、顺序执行 |
| 重复 | 频率（日/周/两周/月/年）、间隔、起算方式 |
| 回顾 | 回顾间隔、下次回顾日期 |
| 提醒 | 截止前提醒、固定时间提醒 |
| 自定义 | 用户自定义字段 |
| 其他 | 备注 |

## 在编辑器里操作任务

块转成任务后，左侧会出现状态圆圈：

- **点击圆圈**：切换状态、设置优先级、加入我的一天、添加提醒、查看属性或移除任务。
- **在面板里左键点击圆圈**：打开状态菜单。
- **在面板里右键任务卡片**：快速改状态、优先级、我的一天、提醒等。
- **点击任务标题**：打开详情弹窗，编辑完整属性。
- **拖拽任务卡片**：在全部任务视图里调整顺序。

## 优先级怎么排

NextAction 会根据重要性、工作量、截止日期紧迫度和手动优先级算出一个分数。分数越高，任务越靠前。

你不需要每次手动拖排序。通常只要把重要性、工作量、截止日填得差不多，列表就能给出一个够用的执行顺序。还没到开始日期的任务不会出现在下一步行动里，避免太早打扰。

相关参数可以在设置里调整。

## 设置

设置面板包含五个部分：

1. **任务默认值**：默认重要性、默认工作量、重建缓存、修复父子关系。
2. **我的一天**：启用开关、每日重置时间、默认视图模式、默认排期时长。
3. **提醒**：启用开关、默认提前量、提示音。
4. **自定义字段**：新增、删除、排序扩展字段。
5. **优先级参数**：截止日、开始日、重要性、衰减、增长、前瞻等参数。

## 安装

**思源集市**：设置 -> 集市 -> 搜索「今天干点啥」-> 安装。

**手动安装**：下载 Release 中的 `siyuan-plugin-nextaction` 文件夹，放入思源工作空间的 `data/plugins/` 目录，然后在「设置 -> 集市 -> 已下载」中启用。

## 开发

```bash
pnpm install
pnpm run dev              # 监听模式：kernel + app 并行
pnpm run build            # 生产构建
pnpm run release          # 构建并部署到本地插件目录
pnpm run release:package  # 构建市场发布用 package.zip
```

架构说明见 [ARCHITECTURE.md](./docs/ARCHITECTURE.md)。

## 发布

发布由 Git tag 触发。确认工作区干净后运行：

```bash
pnpm run release:patch
pnpm run release:minor
pnpm run release:major
pnpm run release:current
pnpm run release:version -- 1.2.3
```

命令会在需要时同步 `package.json` 和 `plugin.json` 的版本号，提交版本变更，创建 `vX.Y.Z` tag，并推送 commit 和 tag。第一次发布时，如果当前版本号已经正确，可以用 `release:current`。GitHub Actions 收到 tag 后会构建 `package.zip` 并创建 GitHub Release。
