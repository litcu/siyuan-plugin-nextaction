# 今天干点啥（NextAction）

一款为 [思源笔记](https://b3log.org/siyuan/) 打造的 GTD 任务管理插件。

不用手动排优先级——插件根据截止日期、重要性、工作量自动算出你最该做什么。任务就住在思源笔记的块上，编辑文档就是在管理任务，不用在两个工具之间来回切换。

<!-- screenshot -->

## 快速上手

1. 从思源集市安装插件，或在 [Release](../../releases) 页面下载后放入工作空间的 `data/plugins/` 目录
2. 在思源中打开任意文档，输入 `/zrw` 把一段文字变成任务
3. 点击块左侧的状态圆圈，切换状态或设置优先级
4. 打开右侧面板的「下一步行动」视图——插件已经帮你排好了

## GTD 工作流

插件围绕 GTD 的五个阶段设计：收集、明确、组织、回顾、执行。

### 收集——把想法快速捕获

- 在编辑器里输入 `/zrw` 把任意块转为任务，任务默认进入**收集箱**状态
- 输入 `/zrxm` 转为项目，`/zrwz` 批量转换（含子块）
- 也可以右键块图标或文档标题图标，选择「转为任务」

收集箱里的任务只是先记下来，不需要马上想清楚怎么做。

### 明确——决定要不要做、怎么做

- 打开**收集箱**视图，逐条处理：不需要的删掉，需要的设好状态、截止日、上下文
- 点击「明确」按钮，任务从 Inbox 进入待开始状态
- 在任务详情弹窗里填写重要性、工作量、上下文（@office、@phone 等）、标签、备注等

### 组织——放进合适的清单

六种任务状态对应 GTD 的不同清单：

| 状态 | 图标 | 用途 |
|------|------|------|
| 收集箱 | 浅蓝圆圈+下箭头 | 刚捕获、还没想清楚 |
| 待开始 | 灰色空心圆 | 明确了但还没动手 |
| 进行中 | 蓝色半填充圆 | 正在做 |
| 等待中 | 橙色虚线圆 | 等别人、等条件 |
| 将来/也许 | 金色圆+三点 | 暂时搁置，以后再说 |
| 已完成 | 绿色实心+白勾 | 做完了 |

五种优先级：紧急（红）> 高（橙）> 中（蓝）> 低（灰）> 无。

任务可以无限嵌套：项目下有任务，任务下有子任务。设置父任务后，子任务未全部完成时父任务不会出现在下一步行动中。

### 回顾——定期检查不遗漏

- **回顾**视图提供 GTD 风格的检查清单，同时列出需要回顾的任务
- 支持给任务设置回顾间隔（天），到时间自动出现在待回顾列表
- 回顾时快速切换状态：把「等待中」变成「待开始」，把「将来/也许」激活

### 执行——专注当下该做的事

- **下一步行动**视图只显示你现在能做的事：已完成、被阻塞、未到开始日期的任务自动过滤掉
- **我的一天**视图做每日规划：从下一步行动里挑选今天的任务，拖到时间线上排期
- 侧边栏面板随时可看：下一步行动、我的一天、收集箱三个 Tab，紧凑显示，带搜索和右键菜单

## 十个视图

| 视图 | 说明 |
|------|------|
| 下一步行动 | 当前可行动的任务，按优先级自动排序 |
| 全部任务 | 所有未完成任务的树形总览，可搜索、筛选、拖拽排序 |
| 项目视图 | 按项目分组，显示子任务和完成进度 |
| 收集箱 | Inbox 状态的任务，快速「明确」 |
| 我的一天 | 今日规划，支持时间线拖拽排期 |
| 将来/也许 | Someday 任务，可「激活」回主清单 |
| 等待中 | Waiting 任务一览 |
| 回顾 | GTD 检查清单 + 待回顾任务 |
| 统计 | 完成率环形图、分布图 |
| 提醒 | 所有待处理提醒 |

## 任务属性

| 类别 | 属性 |
|------|------|
| 基础 | 标题、状态、优先级 |
| 日期 | 截止日、开始日（精确到分钟） |
| 优先级参数 | 重要性（1-7）、工作量（1-7） |
| 组织 | 上下文（@）、标签、父任务、项目 |
| 依赖 | 依赖任务、依赖模式（全部/任一）、顺序执行 |
| 重复 | 频率（日/周/两周/月/年）+ 间隔 + 起始（截止日/完成日） |
| 回顾 | 回顾间隔（天）、下次回顾日期 |
| 提醒 | 截止前提醒（多个提前量）、固定时间提醒 |
| 自定义 | 用户自定义扩展属性 |
| 其他 | 备注 |

## 编辑器中的操作

转为任务后，块左侧显示状态圆圈：

- **点击圆圈**：打开菜单，可切换状态、设置优先级、加入我的一天、添加提醒、查看属性、移除任务
- **面板中左键点击圆圈**：弹出状态选择菜单
- **面板中右键卡片**：快速切换状态、优先级、我的一天、添加提醒、移除任务
- **点击标题**：打开详情弹窗，编辑所有属性
- **拖拽卡片**：排序（全部任务视图）

## 优先级计算

插件采用可配置的乘法公式，根据重要性、工作量、截止日期紧迫度和手动优先级权重计算分数。分数越高的任务越靠前。你只需要设好重要性和截止日，排序交给插件。

开始日期之前的任务不会出现在下一步行动中，避免过早干扰。

## 设置

五个设置 Tab：

1. **任务默认值** — 默认重要性/工作量、重建缓存、修复父子关系
2. **我的一天** — 启用开关、每日重置时间、默认视图模式、默认排期时长
3. **提醒** — 启用开关、默认提前量、音效选择
4. **自定义字段** — 添加/删除/排序自定义属性
5. **优先级参数** — 权重分配、衰减/增长/前瞻等参数

## 安装

**思源集市**：设置 → 集市 → 搜索「今天干点啥」→ 安装

**手动安装**：下载 Release 中的 `siyuan-plugin-nextaction` 文件夹，放入思源工作空间的 `data/plugins/` 目录，然后在 设置 → 集市 → 已下载 中启用。

## 开发

```bash
pnpm install          # 安装依赖
pnpm run dev          # 监听模式（kernel + app 并行）
pnpm run build        # 生产构建
pnpm run release      # 构建 + 部署到本地插件目录
pnpm run release:package  # 构建市场发布用 package.zip
```

技术架构详见 [ARCHITECTURE.md](./docs/ARCHITECTURE.md)。

## 发布

发布由 Git tag 驱动。确认工作区干净后运行以下任一命令：

```bash
pnpm run release:patch
pnpm run release:minor
pnpm run release:major
pnpm run release:current
pnpm run release:version -- 1.2.3
```

命令会在需要时同步更新 `package.json` 和 `plugin.json` 的版本号，提交版本变更，创建 `vX.Y.Z` tag，并推送 commit 与 tag。如果第一次发布时当前版本号已经正确，可以使用 `release:current`。GitHub Actions 收到 tag 后会自动构建 `package.zip` 并创建 GitHub Release。
